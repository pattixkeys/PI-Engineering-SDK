The XKR-32 Rack Mounted KVM is a special X-keys device designed to work specifically with KVMs. In PID #2 the XKR-32 Rack Mounted KVM is only a keyboard, no input or output reports are available thus making it compatible with KVM units. The information below is relevant only to PID #1 which does contain an input and output endpoint. Please note that the XKR-32 Rack Mounted KVM in PID #1 will NOT be compatible with most KVMs, it must operate in PID #2 to be compatible. To convert the device to PID #1 when it is currently in PID #2 hold and press the programming switch located on the right side of the device while plugging it into the usb port. By default the XKR-32 Rack Mounted KVM will return to PID #2, which is the desired PID for use with KVMs, after each reboot however this can be changed using the Reboot Mode output report (described below).

XKR-32 Rack Mounted KVM is supported by P.I. Engineering Macroworks 3.1, X-keys Basic Setup for PC users, our P.I. Engineering SDK for C# .NET, VB .NET, C++, our Linux SDK, and our Mac OS X SDK.

X-keys XKR-32 Rack Mounted KVM Output Report

The following types of output reports are shown in the summary below. These commands are applicable only when the device is in PID #1. Please be aware that several of these commands result in writing to the device's eeprom which has a limit to the number of writes allowed before it is "burnt out". The manufacturer's specification is 50,000 eeprom writes. Because of this we recommend the commands designated with e be executed rarely and not within timing loops. Note, the first byte listed in this documentation is 0 and represents the report ID. This is not present on some non-PC operating systems. So when sending an output report on Android for example, eliminate this byte.

Then after sending this output report a Check Dongle Key input report will be received containing the same 4 bytes returned from the hash if the key matches.
