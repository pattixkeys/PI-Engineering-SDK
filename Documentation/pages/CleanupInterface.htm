CleanupInterface() | Main Page

Overview

Intended for use when device handles have become invalid (for example, when a device has been unplugged). Waits up to half a second for a read attempt in the read thread to fail (which is the normal result of trying to read with an invalid handle), and then does everything that CloseInterface() does, except for closing read and write handles. This function is useful, because closing an invalid handle can cause code to crash or hang.

This function is not available in .NET Framework Class Library.

Unmanaged DLL

CleanupInterface Function

Syntax

VOID __stdcall Cleanup Interface(long hnd);

Parameters

hnd [in]. Handle to desired device, obtained from the EnumeratePIE() call.

Example

See SetDataCallback() example.

Function Information

Minimum DLL Version: PIEHid32.dll
Header: PIEHid32.h
Import Library: PIEHid32.lib
