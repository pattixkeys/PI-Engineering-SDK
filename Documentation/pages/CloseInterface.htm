CloseInterface() | Main Page

Overview

Call this function when you are done using an open device, and are unlikely to use it again. It sets a flag telling the read thread to exit, closes the read handle, forcing the current outstanding read to abort, and waits up to 100 mSec for this to occur. Then it halts the write queue, deletes read and write buffers, and releases system resources (closes mutexes and events). This function should be called before calling SetupInterface() again for this particular interface.

If you load PIEHid32/PIEHid or PIEHid32Net/PIEHidDotNet DLL dynamically, then it is essential to call this function before unloading, for all setup (open) interfaces. In between CloseInterface() and unloading, you must call the Windows API call Sleep, or use some other method of causing a delay, to ensure that no reads are outstanding when the DLL unloads. Failure to follow this advice will normally result in a fatal exception in your executable. Note that a loop is not an effective means of pausing after CloseInterface(), because the idea is to make CPU cycles available to other execution, not to hog the processor.
